Introduction

This is a test book

Exorcist.Spec is a behaviour module that describes how a row of source data should be transformed and persisted to storage. Specs operate on a single row of data.

A Spec describes behaviour: it defines how a row of data should pass through an ETL flow. By itself, a Spec is meaningless until it is given a row to act on. When Exorcist pairs a Spec with a row of data, it spawns a new process and executes the spec's behaviour against the row asynchronously. In Exorcist, this bundle of a Spec and a row is called a Job.
